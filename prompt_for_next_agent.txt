However, during testing, the **HOD 2.0 to 2.0 saving failed on material/texture assignments**. 

**The Root Cause:**
To allow the React UI's material dropdown edits to apply, the previous agent removed the preservation of raw `STAT` chunks and forced `generate_v2_from_model` to dynamically regenerate `STAT` chunks using `model.materials`. 

This works perfectly for HOD 1.0, but HOD 2.0 `STAT` chunks contain a large block of binary shader parameters (glossiness, specular colors, team colors, etc.) immediately *after* the texture maps array. If you look at `parse_stat_material` in `parser/src/hod.rs`, you will see it actually reads these remaining bytes into a `parameters` vector but then throws them away! 

Because the `HODMaterial` struct does not store these parameters, `write_stat_texture_params` builds truncated `STAT` chunks. When the HWRM engine loads these truncated chunks, the shader uniforms are misaligned or missing, causing it to assign wrong textures or corrupt the rendering.
